input.check_headers module¶

input.check_headers.add_missing_headers(to_add=set(), unwanted_uid_filename='unwanted_uid.txt')[source]¶

This function adds the mails that have been missed out, considering the fact that UIDs are consecutive. If a mail that is missing in the JSON file is not available or has been deleted, this function ignores that UID.

Parameters:
  • to_add – A list of UIDs that need to be added. Default value is the list of missing mails’ UIDs.
  • unwanted_uid_filename – The file containing unwanted uids
input.check_headers.check_validity(check_unavailable_uid='False', json_header_filename='headers.json')[source]¶

This function checks for and prints duplicate, missing, and invalid objects in the “headers.json” file. This function can be run first to generate a list of duplicate, missing, or invalid objects’ UIDs which can then be used to add or remove their entries from the JSON file.

Parameters:
  • check_unavailable_uid – If true, prints the unavailable and unwanted uids
  • json_header_filename – The header file to be parsed
Returns:

Last UID that was checked by the function.
input.check_headers.get_unavailable_uid()[source]¶

This function returns a list of UIDs that are not available in the IMAP server.

Returns:List containing the UIDs not available in the IMAP server
input.check_headers.remove_duplicate_headers(to_remove=set(), json_header_filename='headers.json')[source]¶

This function removes all the duplicate entries of the UIDs specified in the to_remove parameter. By default, it removes all the duplicate entries in the JSON file.

Parameters:
  • to_remove – A list of UIDs that need to be removed. Default value is the list of duplicate mails’ UIDs.
  • json_header_filename – The header file from which duplicate entries are removed.
input.check_headers.remove_unwanted_headers(to_remove=set(), json_header_filename='headers.json')[source]¶

This function removes all the UIDs specified in the to_remove parameter. By default, it removes all the unwanted entries in the JSON file, i.e. the list of UIDs of mails that are not forwarded from LKML subscription.

Parameters:
  • to_remove – A list of UIDs that need to be removed. Default value is the list of unwanted mails’ UIDs.
  • json_header_filename – The header file from which unwanted entries are removed.
input.check_headers.replace_invalid_headers(to_replace=set(), json_header_filename='headers.json')[source]¶

This function removes the mail headers that have insufficient attributes and fetches those headers again. If an attribute is missing in the original mail header or if the mail has been deleted, this function ignores that UID.

Parameters:
  • to_replace – A list of UIDs that need to be replaced. Default value is the list of invalid mails’ UIDs.
  • json_header_filename – The json file containing the headers.
input.check_headers.write_uid_map(from_index=1, to_index=0, uid_map_filename='thread_uid_map.json')[source]¶

To ensure that references are correctly recorded in the JSON file such that there are no references to mails that do not exist and to ease the processing of headers, a map with the string in the Message-Id field of the header to the UID of the mail is required. This function fetches the headers from the IMAP server and adds the required pairs of Message_ID and UID to the JSON file.

Parameters:
  • from_index – Fetches headers from this UID onwards.
  • to_index – Fetches headers till this UID (non inclusive).
  • uid_map_filename – The JSON file where the Message_ID-UID mapping is stored.
